The [Definition](/attributes/definition) is the most powerful field on a character. It's also the most freeform, which means it can feel intimidating at first. There's no schema, no rules engine, no checklist.

Think of it less like programming and more like *staging a play*. You're not writing instructions; you're writing examples and context that the character can soak up.

## When you might need it

You probably don't need the Definition if your character is:

- Easy to recognize by name (`Albert Einstein`)
- Doing a generic role (a friendly chef, a calm therapist)
- Conversational, not pattern-based

You probably *do* need it if your character is:

- Playing a structured game (alphabet improv, fortunately/unfortunately, choose-your-own-adventure)
- Speaking in a unique format (all caps, all-rhymes, all-questions)
- Set in a custom world the system doesn't know about
- Generating images and you want consistent visual prompts (see [Direct vs Inferred descriptions](/images/direct-vs-inferred))

## Less is often more

A surprising amount of personality emerges from a great Greeting alone. A bloated Definition can actually hurt, too many examples in conflicting tones make the character feel inconsistent.

Start small. Add one chat. See how it feels. Add another only if you still need it.

<Tip>
  Many of the best characters on the platform have **no Definition at all**. Don't fill it just because it's there.
</Tip>
